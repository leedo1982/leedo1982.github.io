# 도입
프로그래밍을 하면서 생각보다 git commit message를 의미없이 작성하고 있다는 것을 느꼈다.
그래서 이쁜(보기 좋고 이해하기 쉬운) message를 작성하고 싶어졌다.

# 7 Rule
1. 제목과 본문을 빈 행으로 분리한다.
2. 제목 행을 50자로 제한한다.
3. 제목 행 철글자는 대문자로 쓴다.(영문인 경우) 
4. 제목 행 끝에 마침표를 넣지 않는다.
5. 제목 행에 명령문을 사용한다.
6. 본문을 72자 단위로 개행한다.
7. 어떻게 보다는 무엇과 왜를 설명한다.
